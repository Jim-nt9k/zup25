> Краткое описание проекта: что это за конфигурация, для чего предназначена

## Требования

- **Платформа:** 1С:Предприятие 8.3.27+
- **Режим совместимости:** указать версию
- **Вариант работы:** файловый / клиент-серверный
- **Вариант интерфейса:** обычный / управляемый

## Быстрый старт

### 1. Клонирование репозитория

```bash
git clone <URL репозитория>
cd <имя-репозитория>
```

### 2. Загрузка конфигурации в базу

1. Создайте пустую информационную базу
2. Откройте Конфигуратор
3. Меню: **Конфигурация → Загрузить конфигурацию из файлов...**
4. Укажите папку `src/`
5. Обновите конфигурацию базы данных

### 3. Выгрузка изменений в репозиторий

1. Внесите изменения в конфигурацию
2. Меню: **Конфигурация → Выгрузить конфигурацию в файлы...**
3. Укажите папку `src/`
4. Выполните выгрузку

5. Зафиксируйте изменения в git:
```bash
git add .
git commit -m "feat: описание изменений"
git push
```

## Правила работы с репозиторием

### Именование веток

| Тип ветки  |  Формат   |  Пример   |
|------------|-----------|-----------|
| Основная   | `main`    | `main`    |
| Разработка | `develop` | `develop` |
| Фича       | `feature/<номер>-<описание>` | `feature/123-new-report` |
| Исправление| `bugfix/<номер>-<описание>` | `bugfix/456-fix-calculation` |
| Горячее исправление    | `hotfix/<номер>-<описание>` | `hotfix/789-critical-fix` |
| Релиз      | `release/<версия>` | `release/1.2.0.17` |

### Формат коммитов

Используем [Conventional Commits](https://www.conventionalcommits.org/):

Принятый шаблон:
FT-xxxx <тип коммита, см. табл. ниже>: <описание что сделано>

**Типы коммитов:**

| Тип | Описание | Пример |
|-----|----------|--------|
| `feat` | Новая функциональность | `feat(Документы): добавлен документ СчётНаОплату` |
| `fix` | Исправление ошибки | `fix(Отчёты): исправлен расчёт итогов` |
| `docs` | Изменения в документации | `docs: обновлён README` |
| `refactor` | Рефакторинг кода | `refactor(ОбщиеМодули): повторяющийся код вынесен в отдельную функцию` |
| `test` | Добавление тестов | `test: добавлены тесты для СчётНаОплату` |
| `chore` | Прочие изменения | `chore: обновлён .gitignore` |

### Процесс внесения изменений

1. Актуализируйте ветку `develop`:
   ```bash
   git checkout develop
   git pull
   ```

2. Создайте ветку для задачи:
   ```bash
   git checkout -b feature/123-описание
   ```

3. Внесите изменения, выгрузите конфигурацию, зафиксируйте:
   ```bash
   git add .
   git commit -m "feat: описание"
   ```

4. Отправьте ветку и создайте Pull Request:
   ```bash
   git push -u origin feature/123-описание
   ```

5. После ревью — слейте в `develop` (на GitHub)

```bash
# Откат незакоммиченных изменений

# Получение изменений
git pull
```
